## 1. Cursor-based Pagination

এটি আধুনিক এবং স্কেলেবল পদ্ধতি। এখানে কোনো 'Offset' থাকে না, বরং সর্বশেষ দেখা ডেটার একটি ইউনিক আইডি বা মার্কার (Cursor) ব্যবহার করা হয়।

### কাজ করার পদ্ধতি:

এখানে ডাটাবেজকে বলা হয়, "এই ইউনিক আইডির (Cursor) পরের ১০টি ডেটা আমাকে দাও।"

- **Page 1:** প্রথম ১০টি ডেটা দাও। (ধরা যাক শেষ আইটেমের আইডি ১০০)
- **Page 2:** আইডি ১০০ এর পরের ১০টি ডেটা দাও। (ধরা যাক শেষ আইডি ১১৫)
- **Page 3:** আইডি ১১৫ এর পরের ১০টি ডেটা দাও।

## 2. Offset Pagination

এটি সবচেয়ে সহজ এবং বহুল ব্যবহৃত পদ্ধতি। এখানে আমরা বলে দিই যে প্রথম কয়টি ডেটা বাদ যাবে (Offset) এবং এরপর কয়টি ডেটা দেখানো হবে (Limit)।

### কাজ করার পদ্ধতি:

ধরা যাক, প্রতি পেজে ১০টি করে ডেটা দেখাবেন।

- **Page 1:** Offset = 0, Limit = 10 (প্রথম ১০টি)
- **Page 2:** Offset = 10, Limit = 10 (পরের ১০টি)
- **Page 3:** Offset = 20, Limit = 10

### Offset Pagination: সহজ হিসাব

এখানে মাত্র দুটি জিনিস আপনাকে মাথায় রাখতে হবে:

1. **`take`**: আপনি একবারে কয়টি আইটেম দেখতে চান।
2. **`skip`**: কতগুলো আইটেম পার করে বা বাদ দিয়ে পরেরগুলো দেখতে চান।

### ১. সাধারণ কোড স্ট্রাকচার

যখন কোনো কন্ডিশন (Filtering) নেই, তখন কোডটি এমন দেখায়:

```javascript
const pageResults = await prisma.post.findMany({
  skip: 10, // প্রথম ১০টি বাদ দাও
  take: 5, // তারপর ৫টি ডাটা আমাকে দাও
});
```

## ২. বাস্তবে এটি যেভাবে কাজ করে (Real-life Use Case)

ধরুন, আপনার একটা ব্লগ সাইট আছে যেখানে মোট **৫০টি পোস্ট** আছে। আপনি ডিসাইড করলেন প্রতি পেজে **১০টি করে পোস্ট** দেখাবেন।

| পেজ নম্বর  | হিসাব (Formula)                    | Prisma কোড           |
| ---------- | ---------------------------------- | -------------------- |
| **Page 1** | প্রথম থেকে শুরু (কিছু বাদ যাবে না) | `skip: 0, take: 10`  |
| **Page 2** | প্রথম ১০টি বাদ দিন                 | `skip: 10, take: 10` |
| **Page 3** | প্রথম ২০টি বাদ দিন                 | `skip: 20, take: 10` |
| **Page 4** | প্রথম ৩০টি বাদ দিন                 | `skip: 30, take: 10` |

## ৩. ডাইনামিক ভাবে ব্যবহারের নিয়ম

যখন ফ্রন্টএন্ড (যেমন React বা Next.js) থেকে পেজ নম্বর পাঠাবে, তখন ব্যাকএন্ডে হিসাবটা এভাবে করবে:

```javascript
const pageSize = 10; // প্রতি পেজে ১০টি করে
const pageNumber = 3; // ইউজার ৩ নম্বর পেজে ক্লিক করেছে

const results = await prisma.post.findMany({
  skip: (pageNumber - 1) * pageSize, // (3-1) * 10 = 20টি বাদ যাবে
  take: pageSize, // ১০টি ডাটা আসবে
});
```

### কেন এটি ব্যবহার করবেন?

- **সহজ নেভিগেশন:** ইউজার সরাসরি ১ নম্বর পেজ থেকে ৫ নম্বর পেজে ক্লিক করে চলে যেতে পারবে।
- **সিম্পল ইমপ্লিমেন্টেশন:** কোড লেখা এবং লজিক বোঝা খুব সহজ।

### কখন এটি এড়িয়ে চলবেন?

যদি আপনার ডেটাবেসে **লাখ লাখ ডেটা** থাকে, তখন `skip` করতে ডেটাবেসের অনেক সময় লাগে। সেক্ষেত্রে আমরা **Cursor-based pagination** ব্যবহার করি।
